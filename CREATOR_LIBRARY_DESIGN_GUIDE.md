# Creator Library Design Guide

This guide is the operating standard for Kyunolab Creator Library pages.

It documents the current verified structure, generation rules, design principles, and publishing expectations. It does not introduce a new feature, a new page type, or a new URL structure.

Use this guide when creating, editing, reviewing, or regenerating Creator Library pages.

## 1. Definition

Creator Library adapts Kyunolab Archive stories into video production material.

Archive is for reading.

Creator Library is for making.

Creator Library is not a feature only for urban legends. It should support every current Archive category and future categories as the archive expands.

A creator should be able to follow one Creator Library page and complete one video without filming original footage.

## 2. Page Mapping

Create one Creator Library page for one Archive story.

Do not create separate pages for long-form and short-form versions of the same story.

Long-form Creator and Short-form Creator belong on the same Creator Library page.

Do not duplicate categories as long-form categories and short-form categories.

Do not create duplicate pages for the same subject.

## 3. Page Structure

Creator Library pages use this visible order:

1. Story Summary
2. Story Information
3. Creator Toolkit
4. Production Workflow
5. Long-form Creator
6. Short-form Creator
7. Original archive story
8. Reference
9. Related scripts or supporting references

The internal logic is:

1. Story - understand the story.
2. Prepare - prepare production materials.
3. Create - make long-form or short-form video scenes.
4. Finish - review original and reference material.

These internal labels do not need to appear as visible section titles.

## 4. Page And Scene Roles

Page teaches.

Scene produces.

Page-level areas may include:

- production preparation
- Creator Toolkit
- Production Workflow
- narration preparation guidance
- general recording guidance
- tool search direction
- overall production flow

Scene-level areas may include only production outputs:

- Scene Role
- Narration
- Narration Part
- Voice Direction
- Estimated Reading Time
- Scene Focus
- Image Prompt
- Recommended Background Music
- Editing Guide
- Advanced Production

Do not put these inside Scene:

- AI usage instructions
- direct recording instructions
- service recommendations
- pricing notes
- free versus paid comparisons
- installation steps
- production philosophy
- full workflow explanation

## 5. Creator Toolkit

Creator Toolkit is a short Prepare-area overview.

It tells the creator what they may need before making the video. It should not become a tutorial.

Creator Toolkit must cover five balanced areas:

1. Images
2. Narration
3. Background Music
4. Editing
5. Motion (Optional)

Do not make one tool or one medium visually dominant.

Tool names may appear only as examples.

Do not include pricing comparisons, signup prompts, affiliate links, installation steps, or paid-service recommendations.

The user should understand what to prepare within about 30 seconds.

## 6. Production Workflow

Production Workflow explains work order only.

It should stay short and beginner-friendly.

It should not repeat full Creator Toolkit content or explain Scene content in detail.

It should help the creator understand the next action quickly.

Do not add checklist state, login, progress tracking, external links, tutorial video, or ads.

## 7. Long-form Creator

Long-form Creator targets a complete long-form video, generally about 8-10 minutes unless the story naturally requires a slightly different range.

Long-form should not reach length by repeating ideas or adding filler.

Long-form should develop the story step by step.

Each Scene should lead naturally toward the next Scene.

Long-form Creator must be visually distinct from Short-form Creator.

## 8. Short-form Creator

Short-form Creator is not a simple summary of Long-form Creator.

It uses the same source story but should work as its own complete short video.

The first line should create interest quickly.

Reduce unnecessary background explanation.

Short-form Creator must be visually distinct from Long-form Creator.

## 9. Narration

Narration is a spoken video script.

It should work for both AI voice and human reading.

Write narration as speech, not as blog text, academic text, or a report.

Use:

- natural spoken style
- a tone between documentary and storytelling
- short, clear sentences
- one idea per sentence when possible
- line breaks as real reading breaths
- calm tension without forced fear

Avoid:

- blog style
- academic style
- report style
- repeated GPT-like connectors
- overly long sentences
- forced horror
- exaggerated movie-dialogue style
- awkward symbols that may confuse AI TTS

## 10. Narration Part

Long-form Narration may be divided into Narration Parts.

Narration Part is a recording and production unit inside a Scene. It is not a new Scene.

Each Narration Part should generally contain:

- about 2-4 sentences
- about 8-15 seconds of spoken audio
- one clear meaning unit
- no forced break in the middle of a sentence or idea

Short-form Narration should not use Narration Parts unless a Scene becomes unusually long.

## 11. Voice Direction

Voice Direction is short production guidance for both AI voice and direct reading.

Use keyword-style direction such as:

- Calm
- Natural
- Documentary
- Quiet
- Slow Pace
- Slight Tension
- Soft Ending
- Emphasize Final Sentence

Do not write long reading instructions in the basic Scene area.

Do not use service-specific TTS settings or technical audio parameters.

## 12. Scene Workspace

Scene is the smallest production workspace.

A creator should be able to produce one video moment by reading one Scene from top to bottom.

Use this Scene order:

1. Scene number and estimated playback time
2. Scene Role
3. Narration or Narration Part
4. Voice Direction
5. Estimated Reading Time
6. Scene Focus
7. Image Prompt
8. Recommended Background Music
9. Editing Guide
10. Advanced Production

Each production element should be visually distinct and easy to find.

Scene should not feel like a long explanatory document.

## 13. Scene Role

Scene Role tells the creator what the Scene does in the video.

Examples:

- Hook
- Introduce the Story
- Build Suspense
- Present Evidence
- Increase Tension
- Reveal the Mystery
- Closing Reflection
- Leave a Final Question

Scene Role is production information. It is not viewer-facing narration.

## 14. Scene Focus

Scene Focus is a one-line summary of the visual moment.

It appears before Image Prompt.

It does not replace Image Prompt.

It helps the creator understand what the prompt is trying to show.

Do not force the same literal subject into every Scene. Prioritize the Scene's action, background, lighting, composition, mood, and meaning.

When useful, Scene Focus may describe a place, object, person, record, atmosphere, or absence.

## 15. Image Prompt

Image Prompt should be a complete general-purpose English prompt that can be copied into an image generation tool.

Do not use syntax tied to one specific service.

Include only useful visual elements:

- core subject
- action or state
- space and background
- mood
- lighting
- composition
- visual style

Do not force the same character or object into every Scene when doing so weakens the story flow.

## 16. Background Music

Recommended Background Music should provide mood, genre, or search keywords.

Do not recommend a specific paid source.

Prioritize music that does not cover narration.

Do not force a different music direction for every Scene when one consistent atmosphere works better.

## 17. Editing Guide

Editing Guide is not explanation.

It should describe actual screen flow.

Examples:

- Start with a wide shot.
- Slowly zoom toward the subject.
- Hold briefly.
- Fade to black.

Use instructions that a beginner can implement in a basic editor.

Avoid specialist editing settings and complex technical terminology in the basic Scene area.

## 18. Advanced Production

Advanced Production is optional enhancement material.

It must be collapsed by default.

Basic production must be possible without opening it.

Advanced Production may include:

- Motion Prompt
- Sound Effect
- advanced Voice Direction
- Camera and Motion Notes
- Transition and Color Notes
- Negative Prompt

Do not force unnecessary items into Advanced Production.

Do not use syntax tied to a specific paid service.

## 19. Copy Behavior

Copy output should include only content that is useful for production.

Full Long-form Narration copy should include narration text only.

Full Short-form Narration copy should include narration text only.

Narration copy should exclude:

- Scene number
- Scene Role
- Voice Direction
- Estimated Reading Time
- Image Prompt
- Music
- Editing Guide
- page instructions

If item-level copy buttons are added in the future, each should copy only the relevant production body.

Preserve line breaks and paragraph breathing where they help narration.

## 20. Design And Accessibility

Preserve the current Creator Library design language.

Long-form Creator and Short-form Creator must have clear starting points and visual boundaries.

Buttons must use real `button` elements.

Expandable areas must use `aria-expanded` and `aria-controls`.

Keyboard focus must remain visible.

Mobile layout must not overlap, clip, or hide production content.

## 21. Prohibited Content

Do not add:

- pricing comparisons
- free versus paid plan comparisons
- signup prompts
- affiliate links
- professional filming equipment requirements
- advanced filming assumptions
- long service-specific instructions
- long-form and short-form category duplication
- separate long-form and short-form pages for the same topic
- AI usage instructions inside Scenes
- unnecessary repeated information

## 22. New Feature Test

Before adding a new Creator Library feature, confirm:

1. It is needed for actual video production.
2. It helps the creator's next action.
3. It belongs on the Page, not inside Scene, or belongs in Scene as production output.
4. A beginner can understand it.
5. It is not just more information.
6. It does not duplicate the existing structure.
7. It makes production faster or reduces errors.

If it fails these checks, do not add it.

## 23. Gold Standard Representative Pages

Use these current pages as the Gold Standard set:

1. Cursed Images Internet Folklore YouTube Script - internet legend and object/image-centered folklore
2. The Backrooms Explained YouTube Script - place-centered digital folklore and liminal-space production
3. Woman in White Urban Legend YouTube Script - person-centered classic roadside legend
4. Dragon Mythology Around the World YouTube Script - classic mythology and creature comparison

New pages should match this set in:

- section order
- section titles
- Scene Workspace structure
- copy behavior
- collapsed Advanced Production behavior
- Long-form and Short-form separation
- Finish-area reference order

## 24. Completion Checklist

Before finishing Creator Library work, confirm:

1. The page follows the current implemented structure.
2. No unapproved feature or new section was added.
3. This guide does not conflict with Creator Library generation rules.
4. Publishing Pipeline references this guide.
5. New pages can apply the same standard.
6. Links and file paths are correct.
7. Build and existing features are not affected.
